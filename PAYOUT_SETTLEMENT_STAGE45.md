# المرحلة 45 — Driver Payout Settlement

## النطاق
لوحة قراءة‑فقط `/payout-settlement` تبني دفعة تسوية مقترحة للسحوبات قيد التنفيذ (PENDING/APPROVED)، تقيس تغطية كل طلب بصافي أرباح السائق المتاح، وتُوصي بالإجراء المناسب. تبني مباشرة على المرحلة 44 (نزاهة الدفعات).

## Backend (إضافي — متوافق خلفيًا)
- `GET /withdrawals/settlement-proposal?limit` (`payments.read` / `payments.manage`) — `WithdrawalsService.settlementProposal`.
- يجلب السحوبات PENDING/APPROVED (الأقدم أولاً) + صافي أرباح كل سائق + إجمالي المدفوع.
- لكل سائق رصيد قابل للتغطية = net − paid، يُستهلك تدريجيًا عبر الطلبات.
- التوصيات: `APPROVE` (جاهز للاعتماد)، `PAY` (جاهز للدفع)، `REVIEW_RISK`، `HOLD_INSUFFICIENT`.
- إشارات: `INSUFFICIENT_FUNDS`، `DRIVER_RISK`.
- لا تحريك أموال ولا تعديل مخطط؛ التنفيذ يتم عبر مسارات `approve`/`markPaid` المحكومة بـ `payments.manage` مع AuditLog.

## الواجهة
- بطاقات إجمالية (طلبات/مبلغ/جاهز للاعتماد/جاهز للدفع/محجوز) + جدول توصيات + فلتر (الكل/جاهز/محجوز) + تحديث تلقائي كل 30 ثانية.
- روابط إلى نزاهة الدفعات وطلبات السحب للإجراء.

## الحوكمة
قراءة فقط؛ Ledger يبقى مصدر الحقيقة؛ أي تحريك مالي يمرّ حصرًا عبر تحويلات السحب المدقّقة.

هذه الحزمة تنفّذ Stage45 فقط ولا تبدأ Stage46.
